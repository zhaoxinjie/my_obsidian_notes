# 个人知识库 Agent 维护指南

这个 vault 采用 `LLM Wiki` 模式：`raw` 保存原始资料，`wiki` 是持续维护的知识层，这个文件定义 LLM agent 应该如何工作。

## 目标

帮助用户在 Obsidian 中持续建设和维护个人知识库。

人负责选择资料、提出问题、决定重点。
Agent 负责阅读、整理、串联、更新与维护。

## Vault 结构

- `raw/`: 不可随意改动的原始资料与收件箱
- `wiki/`: 由 LLM 持续维护的知识页面
- `system/`: 模板、工作流与维护说明
- 根目录笔记：导航与上手入口

## 基本规则

1. 除非用户明确要求，否则不要修改 `raw/sources/` 下的文件。
2. 把 `raw/inbox/` 视为暂存区：可以分类、重命名、建立链接，但原始内容应尽量保持不变。
3. `wiki` 是编译后的知识层。优先更新已有页面，而不是重复新建。
4. 每次有意义的摄入、问答沉淀或维护操作，都必须更新 `wiki/index.md`，并向 `wiki/log.md` 追加记录。
5. 充分使用 Obsidian 双链 `[[Page Name]]`。这套系统的价值来自页面之间的连接。
6. 当新来源改变已有理解时，要同步修订相关页面，并明确标注矛盾、不确定性或版本变化。
7. 保留来源链路。总结页应链接回相关来源页；必要时标注原始文件路径。
8. 页面应便于快速浏览：优先使用标题、要点、短段落与“开放问题”小节。

## 标准页面类型

- `wiki/overview.md`: 知识库总览页
- `wiki/index.md`: 知识页目录
- `wiki/log.md`: 按时间追加的活动日志
- `wiki/sources/`: 每个已摄入来源对应一页
- `wiki/entities/`: 人物、公司、书籍、工具、地点、项目等实体页
- `wiki/concepts/`: 主题、概念、框架、长期问题域
- `wiki/questions/`: 值得长期保留的问题与回答
- `wiki/syntheses/`: 比较、分析、主题综述、多来源综合页

## 标准工作流

### 1. 摄入新来源

当用户要求摄入一份来源时：

1. 从 `raw/inbox/` 或 `raw/sources/` 读取来源。
2. 识别：
   - 核心主张
   - 重要实体
   - 重要概念
   - 开放问题
   - 它如何改变当前 wiki 的理解
3. 创建或更新：
   - `wiki/sources/` 中的一张来源页
   - 受影响的实体页
   - 受影响的概念页
   - 如果它显著改变了更大主题，则更新或新增综合页
4. 更新 `wiki/index.md`。
5. 向 `wiki/log.md` 追加记录。

### 2. 回答问题

当用户提出问题时：

1. 先阅读 `wiki/index.md`。
2. 打开最相关的 wiki 页面。
3. 基于 wiki 合成回答。
4. 如果答案具有长期价值，则沉淀进 `wiki/questions/` 或 `wiki/syntheses/`。
5. 如果产生了新页面，同步更新 `wiki/index.md` 和 `wiki/log.md`。

### 3. 巡检知识库

当用户要求健康检查时：

重点检查：

- 孤立页
- 重复概念
- 过时结论
- 链接稀疏或关联不足
- 缺失的来源页
- 尚未解决的矛盾
- 应该拆分或合并的页面

把发现写入 `wiki/log.md`，并更新相关页面。

## Frontmatter 约定

在确有帮助时使用 YAML frontmatter。推荐字段：

- `type`: source | entity | concept | question | synthesis | overview
- `status`: seed | active | stable | needs-review
- `created`: YYYY-MM-DD
- `updated`: YYYY-MM-DD
- `source_count`: 适用时填写整数
- `tags`: 简短且有用的标签列表

不要添加无助于导航或筛选的噪声元数据。

## 命名约定

- 使用清晰、稳定的标题。
- 实体页和概念页优先使用单数形式或稳定命名。
- 来源页在日期明确时应以前缀日期命名，例如：
  - `2026-04-21 Karpathy - LLM Wiki`
- 问题页应写成自然语言问题。
- 综合页应写成可复用笔记，而不是聊天记录。

## 写作风格

- wiki 页面默认必须使用中文表达。
- 可以在首次出现的重要术语后保留英文注释，例如：`智能体（agent）`、`感知（perception）`。
- 页面标题、小节标题、说明文字优先使用中文；除非原始专有名词不适合翻译，否则不要直接用英文做主标题。
- 原始资料标题在有帮助时保留原语言。
- 明确区分事实、解释与推测。
- 优先写出六个月后再看仍然成立的总结。

## 避免事项

- 不要到处堆砌原文摘录。
- 如果已有页面只需要新增一个小节，就不要另起新页。
- 不要只做来源页而不把它整合进更大的 wiki 网络。
- 如果某个主题本应已有维护页，不要只根据原始资料临时回答。

## 推荐提示方式

用户可能会这样说：

- "把 `raw/inbox/` 里的这篇文章摄入知识库"
- "根据现有 wiki，回答我关于 X 的问题，并把答案存档"
- "帮我给这个知识库做一次 lint，找出断链、重复页和过时观点"
- "把最近 5 条 log 相关的内容整理成一个主题总结"

遵循以上工作流，维护好页面之间的一致性与链接关系。
