# 使用手册

## 这套系统怎么理解

你以后不需要自己手工维护大量笔记。

你的主要动作只有三个：

1. 收集资料
2. 提问题
3. 让 LLM 帮你整理和维护

## 日常工作流

### 1. 收资料

把下面这些内容放进 `raw/inbox/`：

- 剪藏的网页
- 阅读笔记
- 会议纪要
- 项目记录
- 灵感草稿

### 2. 摄入资料

直接对我说：

```text
把 raw/inbox/这篇文章.md 摄入知识库
```

或：

```text
帮我处理 raw/inbox/ 里最新的 3 篇资料
```

我会把资料整理进 wiki，而不是只在聊天里给你一段总结。

### 3. 追问和沉淀

你可以继续问：

```text
根据现在的知识库，这篇资料改变了我对 AI 产品设计的什么看法？
```

```text
把我关于“个人知识管理”的现有观点整理成一页综合分析
```

```text
比较一下我知识库里关于长期主义和快速试错的张力
```

如果答案值得保留，我会存进 `wiki/questions/` 或 `wiki/syntheses/`。

### 4. 定期巡检

建议每周或每两周做一次：

```text
给这个知识库做一次 lint
```

我会检查：

- 重复页
- 孤立页
- 断链
- 过时判断
- 缺少来源支撑的结论

## 最推荐的三种使用场景

### 主题研究

比如你长期研究：

- AI
- 产品管理
- 写作
- 健康
- 投资

每来一篇资料就摄入一次，几周后就会形成一张越来越清晰的知识网络。

### 个人操作系统

把这些内容逐渐放进来：

- 周报
- 月复盘
- 目标拆解
- 决策记录
- 习惯观察

这样知识库会慢慢变成你的“外部大脑”。

### 项目知识库

任何长期项目都可以建立对应的 entity/concept/synthesis 页面。

比如：

- 某个副业项目
- 某个内容选题
- 某个学习计划

## 推荐你对我说的话

- "帮我摄入这篇资料，并更新相关主题页"
- "根据现有 wiki 回答这个问题，并把答案存档"
- "给我的知识库做一次结构整理"
- "看看最近有哪些主题正在浮现"
- "把这个主题做成一页可长期维护的概念页"

## 第一周建议

1. 放入 3 到 5 篇你最近最重要的资料
2. 让我逐篇摄入
3. 选一个你最关心的主题做第一张概念页
4. 再做一次 lint

这样你很快就会看到它和普通“资料堆积”之间的区别。
